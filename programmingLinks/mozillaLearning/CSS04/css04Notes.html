CSS Layout Notes

    Flexbox

  • Flexbox: a method of displaying elements on a webpage that allows for greater formatting options than those available to block and inline only. For example, flexbox allows three articles to be ordered one next to the other, with each taking up the most of their available space in instead of only the space their content takes up.
  • To make something use the flex display, set its parent element to display: flex;. This will cause each child of that element to display as flex elements.
  • Elements laid out as flexible boxes are laid along two axes: main axis and cross axis.
  • flex-wrap: wrap; use inside flexbox container to allow flex items to wrap below each other.
  • flex-basis: xxpx; use to set how wide a flex item will be before it is moved to a new line. Applied to flex items, not flex containers.
  • flex-grow: xx Used as a unitless value to dicate what proportion of the line each flex item will use. If different items are given unique values, they will take up more/less of the line as appropriate. This can also have another unit given, as before: flex: 1 200px; Here, each flex item will have a maximum of 200px, then any remaining space will be divided between each flex item according to their proportions.
  • The flex: shorthand sets, in order: flex-grow, flex-shrink, flex-basis.
  • flex-flow: row wrap shorthand for flex-wrap and flex-direction: (row/column(default)).
  • aling-items: (stretch/center/start/end/flex-start/flex-end/more-available); controls where flex items align on the cross axis.Default value is stretch.
  • justify-content: (start/center/space-around/space-between/space-evenly) determines where the flex items lay on the main axis. Default value is flex-start.
  • order: x sets the position of a flex item. By default, all flex items have an order value of 0. Those with a higher value appear later. Negative values will cause the item to appear earlier. Flex items with the same order value will appear in their source order.

    • Grids

  • Grids allow us to organize our page into columns, rows, and the spaces between (normally called gutters), and keep this layout consistent among pages with different content. We use display: grid; to set our layout to grid, just like setting our layout to flex.
  • Set the number of columns used grid-template-columns: xxpx xxpx xxpx ..., using px or a percentage, and declaring the width for as many columns as you need. Also available is the fr unit, which gives each column a portion of the width equal to the part of the whole the value it is given is. For example: grid-template-columns: 1fr 1fr 2fr; would give each of the first two columns one quarter of the total width (1/4) and the last column half the total width (2/4, 1/2). The number of columns can be repeated using grid-template-columns: repeat(x, y); where x is the number of times that y should be repeated. So grid-template-columns: repeat(2, 1fr); would repeat a listing twice with 1fr of the width of the row before moving to the next row. Similarly, grid-template-columns: repeat(3, 200px) would have three listings per row at 200px wide each, before moving to the next row.

  • Gaps for spaces between rows and columns we use grid-column-gap, grid-row-gap, or grid-gap. Gap values can be lengths or percentages, but not fr. While grid-gap is still supported, gap will be the new standard, and so both should be used to maintain the code for both old and new browsers.
  • Implicit and Explicit Grids Implicit grids occur when we have not declared anything for one axis of grid-template-(columns/rows). Explicit grids are for when we have declared a value for either axis of the same. Implicit grids automatically adjust their width or height to fit the content, but can be given values with grid-auto-(rows/columns). This will ensure that the columns or rows given that value are always that value.
  • grid-auto-(rows/columns): minmax(x,y); IMPORTANT! here, and below! This will allow the axis to which it is set to have a minimum size value of x and a maximum size value of y, based on the content inside the content box (see the first box in the example grid above). This would have solved the issue of content flowing out of the buttons on the Knave character generator.
  • grid-template-(columns/rows): repeat(auto-fill,minmax(x, 1fr)); This sets the size of the axis to a minimum width of x, and will expand as needed to fill the available space. This is different from using only minmax because it will add rows/colums as needed to make space for the contant to fit, not just adjust the width/height of the box it is in.

  • The position property is what we use to set the position of elements in the page. It accepts static/relative/absolute/sticky property values.
    • static is the default position of all elements.
    • relative is almost the same as static, but it allows for further positioning that lets the element overlap other elements. Use the top/bottom/left/right: xxpx; properties to change a relative positioned element's position.
    • absolute remove the element from normal flow entirely. The element sits ontop other elements on its own layer. top/bottom/left/right: xxpx; works differently with absolute positioning: it specifies how far the absolute positioned element should be from each of the containing element's sides.
      • The container the absolute element is positioned by is usually the HTML document. However, by changing the position value of one of the elements the absolute element sits inside, we can cause the absolute element to be positioned based on the changed element's position. For example, in the example above, changing the position of the article the absolute paragraph is inside of to position: relative;, the absolute positioned paragraph with be positioned based on the article, not the HTML document.
      • z-index: x; The z-index is used to set the plane on which elements appear. Negative values are farther from the top (toward the back of the monitor), and positive values are farther above the top (the front of the monitor). Z-index can be used to better control how elements stack ontop each other when using positioning techniques like absolute. z-index values are normally set to auto. By default, elements farther down the page appear ontop of elements before them.
    • fixed is used for fixing an element to a specific area of the page regardless of where in the page the viewport is. The element is positioned relative to the screen. At least one top/bottom/left/right value is required for this position type. margin: auto 0; can be used to center the element. Example code for this is in this page's CSS under .positionExample .fixedHeading, but the code is commented out for the sake of the rest of the page's clarity.
    • sticky is similar to fixed, but allows for the element to only stick to a part of the page under certain conditions. The most common use for this is would be creating an elemnt to be stuck to the page once it has been reached in the viewport, like the letter headings to an index (think <dt> elements staying at the top of the page until the next one is reached in a dictionary). The code for such an element might look like so: 
      dt {
background-color: black;
    position: sticky;
    top: 0;
    left: 0;
    margin: 1em 0;
}

  • Multi column layouts use the column-count/column-width properties. The column-count property accepts a unitless value and separates the content in the element into that many columns. For example : column-count: 3; would separate the content into 3 columns.
  • column-width: xxpx; fits as many columns of the provided width into the element as it can. For example, a 500px wide container with column-width: 200px would have 2 200px wide columns in it.
  • Columns may not be styled individually, but two properties exist to style the set: column-gap: xxpx; sets the distance between columns, and column-rule: width style color; sets a border-like rule between columns. Note: column-rule does not create space in the column gap, it simply lays ontop of it.
  • Fragmentation in Columns: This seems like a niche issue. Basically, if you have content in columns, and the content becomes separated from itself due to the column ending (so the content flows to the next column), Mozilla thinks this might look bad. You can fix it using the page-break: avoid inside the element you do not want to break. Also include page-break-inside: avoid; for legacy support. Note: I'm not convinced this would look bad on a page with a multi column layout that does not scroll. If the page does scroll, the separation of content might be confusing for the reader, but.. I'm not convinced.

    • Responsive Design

  • Responsive Design is a term used for the idea that websites should be built for a variety of devices, and respond to the device's size or screen resolution appropriately.
  • Mobile First Design is the concept of designing a page in one column first, so that it may be used for mobile devices. After that design is finished, the site is designed for larger devices.
  • Text Sized Based on the Viewport The size unit vw is based on the size of the viewport. While one should not use this unit alone (it disregards the user zooming the text and keeps it one size), it is suggested it be combined with other size units like px or em. Doing so allows the text to scale with the viewport while remaining zoomable. Example: font-size: calc(1.5em+3vw);
  • Viewport Meta Tag A very important tag in the <meta> element of an HTML document. It will look like this: <meta name="viewport" content="width=device-width,initial-scale=1">. This allows the website to respond to the actual size of the device's viewport, not that set by the device. Read more about why this is on Mozilla here.

    • Media Queries

  • Media queries have three parts:
    • Media Type Tells the browser what media type the code is for (all, screen, print, speech).
    • Media Expression Rule that determines if the CSS is applied.
    • CSS rules that are applied if the media expression test is passed.
  • Here are some common media queries:
    • min/max-width/height Usually we use width for this, not height. Helpful for changing the site's layout based on how large the user's screen is. For example, to use CSS specific to screens less wide than 360px: @media screen (max-width: 360px){...}.
    • orientation Used to figure out if the device is in landscape or portrait mode.
    • hover Used to figure out if the device has the ability to hover. If it does not, it is probably a touch device. Example: @media (hover: hover){...}.
    • pointer Inquires about the devices pointer input capability: none, fine, course. None means they cannot point (no mouse or touchscreen), fine means they can use a touchscreen, and course means they have a mouse or trackpad. This is a great way to design for mobile phones or tablets, as you are able to make selectable elements larger for the user, or have a different way to interact with the page entirely.
  • Advanced Media Queries We can use logic to combine media queries for more complicated queries. and checks if more than one media query is true. Or checks to see if one of many are true and uses a comma to separate queries. not check to see if the query is not true and applies to CSS if it is not. Here are some examples: 
                    
@media screen and (min-width: 400px) and (min-height: 300px){...}

@media screen and (min-width: 400px), (orientation: portrait){...}

@media screen not all and (min-width: 400px) and (orientation: landscape){...}
