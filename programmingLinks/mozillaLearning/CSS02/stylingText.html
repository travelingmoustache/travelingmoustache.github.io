Styling Text

Styling text falls into two categories: font style and text layout style. Font style changes properties of the font like which font is used, whether it is bold or italic, and more. Text layout style changes the spacing of the font and how it is aligned within the page.

Font Style

color
The color property determines the color of the font.
font-family
The font-family property changes which fonts are used for the text in the order they are given in the font stack. The browser will display the text in the first font in the font stack it has access to. If none of the fonts are available, the browser will use one of its default fonts. An example of the font-family property and a font stack: font-family: "Trebuchet MS", Veranda, sans-serif;.
Web Safe Fonts
Web safe fonts are the fonts that any machine should be able to display. They can be read about on Mozilla's Styling Text tutorial here.
Default Fonts
Default fonts are the fonts any browser can reliably display. If all else fails, the browser will display whichever default font it is instructed to: serif, sans-serif, monospace, cursive, or fantasy. The default font that closely matches the intended font should always be given at the end of the font stack.
font-size
font-size determines the size of the characters displayed on the page. This value is usually measured in px, em, or rem. The default size for all browsers is 16px. This value can be changed on a per-element basis. The em property value is determined by the defualt font-size of the element it is in: 1em=its elements default font-size. So, to get 24px sized font on a page with an unmodified default font-size, you would use 1.5em. This ensure the text will always be 1.5 times the size of the default font-size for that element, regardless of whether or not it changes later. rem is the same, except it uses the root default font-size set for that page, not for the element in which it is contained. So by setting the pages default font-size to something easy like 10px, the maths for sizing using rem becomes much easier.
font-style
Used for modifying italics. Property values include normal, italic, and oblique. Normal resets the text to normal (if it previously had style applied to it). Oblique slants the font in an attempt to immitate italic text.
font-weight
Sets the boldness of the text. Property values include normal, bold (the two most commonly used), lighter and bolder (which set the boldness one step either direction from its current boldness), and 100-900 (which give a finer-grained tuning ability to text boldness).
text-transform
Transform the text. Property values include none (remove or prevent transformations), uppercase and lowercase, capitalize (capitalizes all text), or full-width (gives each character a set width that can be aligned with multiple languages).
text-decoration
Sets lines through fonts. Available property values include none, underline, overline, and line-through. Text decoration can accept multiple values, like so: text-decoration: underline overline line-through wavy black.
text-shadow
This has a lot to it and I'm not interested in taking all those notes; I don't see myself using this much. Mozilla's tutorial on it can be found here.

Text Layout

text-align
Used to determine the alignment of the text within its containing content box. Includes left, right, and center property values, as well as justify. Justify causes each line to be the same width, like most printed books are. This will create spaces in text to make sure their lines are the same width, sometimes causing lines with long words to look strange, and should therefore be used carefully.
line-height
Determines the height of each line. While it can accept traditional value measurements like px and em, because it is based on the size of the text, it is best to use a unitless value that is a multiplier of the text-size. Most pages used a line-height of 1.5-2, making the text much more readable.
line-spacing and word-spacing
Fairly self explanatory. Won't be used often, but might be utilized for a certain look.

Other Font Properties

Available through Mozilla here is a list of more properties used for styling text.

Font shorthand is available as well. The order for property values is given as: font-style, font-variant, font-weight, font-stretch, font-size, line-height, font-family. The two property values required are font-size and font-family, and a forward slash is required between the font-size and line-height property values. Here is an example: font: italic normal bold normal 3em/1.5 Helvetica, Arial, sans-serif;.

Styling Lists

Vertical Rhythm
Vertical rhythm describes the way the page flows vertically, including the size of text and spacing of lines. Lists should keep the same line-spacing that the text on the page has. This can be set easily with something line p, ol, ul, dl: line-height 1.5em; which would cause each list type, and the children of those list types, to have line-spacing consistent with the paragraph element.
List-Specific Styles
list-style-type:
Sets the type of bullet used for the list item. For example: list-style-type: upper-roman;.
list-style-position
Sets whether the bullets appear outside the list item or inside the list with the item; accepts inside or outside property values.
list-style-image
Allows the use of a custom image as a bullet. For example: list-style-image: url(example.svg);.
List Style Shorthand
The shorthand for list styles uses the properties above in any order. For example: list-style: square url(star.svg) inside;
List Counting
start
The start attribute determines what number the list will start counting at. For example: <ul start="4">.
reverse
The reverse attribute will cause the list to count in reverse. No values are needed for this attribute.
value
The value attribute allows you to set values for list items manually, such as: <li value="2">. This would set that list item's value to 2.

Styling Links

Link States

Link (Unvisited)
Styling for a standard, unvisited, unfocused (etc.) link. Styled using the :link pseudo class.
Visited
Styling for a link that's been visited, whose URL exists in the browser's history. Styled using the pseudo class :visited.
Hover
Styling for a link that's being hovered over by the mouse. Styled using the pseudo class :hover.
Focus
Styling for a link that's being focused by the keyboard or programmatically. Styled using the pseudo class :focus.
Active
Styling for a link that's being actively engaged with. Styled using the pseudo class :active.

Icons on Links

Icons can be included with links, just as they are with bulleted lists or anything else. We can find where to place icons for links by using a[href*="http"]{ }, which will select any <a> element with an href attribute whose value starts with http. Then, using the background, background-size, and padding-right properties, we can select the image URL, set its width and height in pixels, and add padding to keep it aligined with the text it is next to. Web Fonts

Web fonts use fonts available on the web instead of those only available on one's local machine. Web fonts are specified using the @font-face { } block at the start of a style sheet.

Using Web Fonts

  • Download a font from a site like Font Font Squirrel. Make sure the license allows commercial use.
  • Go to Font Squirrel's web font generator, upload the font to be used, and download the package created by this site. This will provide various versions of the font to be used across web browsers, format the CSS, and provide an example HTML page to see what the font might look like.
  • Open the provided CSS file (stylesheet.css), and copy the code given into the very top of the stylesheet for your site. Now these fonts may be accessed by your site.
    • Note: make sure the URLs that point to the font files in the stylesheet are pointing to the right location.
  • Access fonts the same way as any other font: font-family: 'acknowledgementmedium', serif;.

Using Google Fonts

  • Go to the Google Fonts site.
  • Add fonts to your package by clicking the red plus sign on the font. These fonts will be collected at the bottom of the page under family selected.
  • Copy the HTML given at the top of the family selected box and paste it into your HTML document, above where the stylesheet is linked.
  • Apply the custom fonts as you would any other, by copying the text given under specify in CSS.
